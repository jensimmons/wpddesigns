How to contribute to Web Platform Docs

Step 1: Join the Group Email List.

The first step is to sign up for our email group. Join the list by emailing an empty message with the subject “subscribe” to public-webplatform-request@w3.org from the email address you'd like to use. You can find more, and read the archives, at lists.w3.org/Archives/Public/public-webplatform.

Step 2: Introduce Yourself.

Once you are on the list, please email the group and introduce yourself. Who are you? What would you like to contribute?

Step 3: Jump into chat.

To talk to folks in real time, and get a quick answer to a question, jump into our public chat room. We are using IRC — a type of chatroom that's older than the web. We have a dedicated chatroom (called a “channel”) on the IRC network Freenode.

Server: Freenode (irc.freenode.net)
Channel: #webplatform

Please feel free to enter the IRC channel anytime, and just start talking to whoever’s there. You can connect through an IRC client (standalone software) or this web interface. Here’s more on how to get started.

Get a Media Wiki Account and Check Out The Editor’s Guide

Much of the work that’s needed revolves around editing the docs themselves. Currently, all the content about HTML, CSS, Javascript and more is on a Media Wiki website. Anyone can sign-up for an account, which allows you to log in and edit pages. Find information about how to get an account and what work is needed to be done at docs.webplatform.org/wiki/WPD:Editors_Guide.

Write Javascript Documentation with Web Platform Wednesdays

“Web Platform Wednesdays” is a weekly effort to organize who’s going to do what for the week. A small group focuses together on one particular section of the site, to gain momentum and get that section done. In 2013, the group focused on making a first pass through all the CSS documentation, creating a usable page for each property and attribute. Now Web Platform Wednesdays is changing focus to JavaScript. If you are interested in helping to write JavaScript documentation, please do join in. Email the list and let us know you want to be involved. You can see the current list of who is editing which page at: docs.webplatform.org/wiki/Meta:web_platform_wednesday.

Attend (or Host) a Doc Sprint.

A great way to join the effort is to attend a doc sprint. Organized by different people around the world, WebPlatform.org doc sprints rather a large group of people, in person, to spend the day writing documentation. We’ll help you get setup, walk you through the process, and show you where to start.

To find out if there’s an upcoming doc sprint near you, check the list at docs.webplatform.org/wiki/WPD:Doc_Sprint. If you are interested in organizing a doc sprint in your area, email the list and let us know.

Not Yet Another Reinvention Of The Wheel

Where WPDocs are Born

There are a lot of great resources out there for learning about the web. Each page on WPD started out as a page on MSDN. Microsoft generously donated their content to get us started. MDN is a favorite among professional developers for looking up reference docs about the web. Mozilla has been generously allowing volunteers to copy content from MDN to WPD, helping us improve each page. And much of the content on WPD comes from the specifications themselves. Web standards specifications are written by experts to specify how browser makers should build their browsers. WPD will grow into the place for future spec writers to document their spec for developers. Easier for devs to understand and find the information their need.

Mashing together information from various sources takes time, however. Scripts have gotten us started, but then we need real people to write or rewrite each page to make it useful.
